## 🚀 Getting Started

### **Welcome to DFCMS!**
The Digital Feedback & Complaint Management System is your official channel for submitting feedback, complaints, and grievances. This guide will help you navigate the system effectively.

### **First-Time Login**
1. **Visit**: [https://dfcms.university.edu](https://dfcms.university.edu)
2. **Click**: "Login" button
3. **Enter**: Your university email and password
4. **Complete**: Two-factor authentication (if enabled)

### **System Requirements**
- **Browser**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **Internet**: Stable connection (minimum 2 Mbps)
- **Device**: Desktop, tablet, or smartphone
- **JavaScript**: Must be enabled

## 📝 Submitting Complaints

### **Step-by-Step Guide**

#### **1. Access Complaint Form**
1. Click **"➕ New Complaint"** on dashboard
2. Or navigate to **Complaints → Submit New**

#### **2. Fill Complaint Details**

##### **Basic Information**
```
📋 Complaint Details
┌─────────────────────────────────────────┐
│ Category: [Academic ▼]                  │
│ Priority: [High ▼]                      │
│ Subject: [Issue with Lab Equipment]     │
│                                         │
│ Description:                            │
│ ┌─────────────────────────────────────┐ │
│ │ The oscilloscope in Lab 204 is not  │ │
│ │ functioning properly. Display is    │ │
│ │ flickering and measurements are     │ │
│ │ inaccurate. This affects our        │ │
│ │ practical sessions.                 │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ Attachments: [+ Add Files]             │
└─────────────────────────────────────────┘
```

##### **Category Options**
- **Academic**: Course-related issues, grades, materials
- **Administrative**: Registration, fees, documentation
- **Technical**: Equipment, software, network issues
- **Facilities**: Building maintenance, cleanliness
- **Safety**: Security, health concerns
- **Other**: General feedback and suggestions

##### **Priority Levels**
- **🔴 High**: Urgent issues affecting studies/safety
- **🟡 Medium**: Important but not urgent
- **🟢 Low**: Minor issues or suggestions

#### **3. Add Attachments (Optional)**
- **Supported Formats**: PDF, DOC, DOCX, JPG, PNG
- **Maximum Size**: 10MB per file
- **Maximum Files**: 5 attachments per complaint

#### **4. Review and Submit**
```
📋 Review Complaint
┌─────────────────────────────────────────┐
│ Category: Academic                      │
│ Priority: High                          │
│ Subject: Issue with Lab Equipment       │
│                                         │
│ This complaint will be sent to:         │
│ • Class Representative (CR)             │
│ • Department: Computer Science          │
│                                         │
│ Estimated response time: 24-48 hours    │
│                                         │
│ [Submit Complaint] [Save as Draft]     │
└─────────────────────────────────────────┘
```

### **Best Practices for Submitting Complaints**

#### **✅ Do's**
- **Be specific** in your description
- **Include relevant details** (dates, times, locations)
- **Attach evidence** (photos, documents)
- **Choose appropriate category** and priority
- **Use professional language**
- **Submit one issue per complaint**

#### **❌ Don'ts**
- **Don't submit duplicate complaints**
- **Don't use offensive language**
- **Don't include personal information** of others
- **Don't exaggerate issues**
- **Don't submit non-university related complaints**

## 📊 Tracking Complaints

### **Complaint Status Flow**

```
📝 Submitted → 🔄 Under Review → ⚙️ In Progress → ✅ Resolved
      ↓              ↓                ↓              ↓
   (Auto)        (CR/Teacher)    (Assigned)     (Completed)
```

#### **Status Descriptions**

| Status | Meaning | Typical Duration |
|--------|---------|------------------|
| **Submitted** | Complaint received and queued | < 1 hour |
| **Under Review** | Initial assessment by CR/Teacher | 1-24 hours |
| **In Progress** | Active investigation/action | 1-7 days |
| **Resolved** | Issue successfully addressed | - |
| **Rejected** | Complaint not valid/applicable | - |

## ❓ Frequently Asked Questions

### **General Questions**

#### **Q: How do I know my complaint was received?**
**A**: You'll receive an immediate confirmation email with your complaint ID. The complaint will also appear in your dashboard within minutes.

#### **Q: What's the average response time?**
**A**: 
- **High Priority**: 4-6 hours
- **Medium Priority**: 24-48 hours
- **Low Priority**: 3-5 days

#### **Q: Can I submit anonymous complaints?**
**A**: No, all complaints require identification for proper follow-up and resolution.

#### **Q: What if I'm not satisfied with the resolution?**
**A**: You can request a review within 7 days of resolution. Click "Request Review" in the complaint details.

### **Technical Questions**

#### **Q: I forgot my password. How do I reset it?**
**A**: 
1. Click "Forgot Password" on login page
2. Enter your university email
3. Check email for reset link
4. Create new password following security requirements

#### **Q: Why can't I upload files?**
**A**: Check:
- File size is under 10MB
- File format is supported (PDF, DOC, JPG, PNG)
- You have stable internet connection
- Browser is updated to latest version

#### **Q: The system seems slow. What should I do?**
**A**: 
1. Clear browser cache and cookies
2. Try a different browser
3. Check internet connection speed
4. Contact IT support if issues persist

### **Process Questions**

#### **Q: Can I edit my complaint after submission?**
**A**: You can add comments and attachments, but cannot change the original complaint details. Contact the handler if major corrections are needed.

#### **Q: How long are complaints kept in the system?**
**A**: All complaints are retained for 7 years as per university policy and legal requirements.

#### **Q: Can I submit complaints about faculty members?**
**A**: Yes, academic-related complaints are handled through the same system with appropriate confidentiality measures.

### **Privacy Questions**

#### **Q: Who can see my complaints?**
**A**: Only authorized personnel based on the complaint category:
- **Class Representatives** for student-related issues
- **Teachers** for academic matters
- **Department Heads** for escalated issues
- **Lab Assistants** for technical problems

#### **Q: Is my personal information secure?**
**A**: Yes, all data is encrypted and protected according to university security policies and GDPR compliance.

## 📈 Tips for Success

### **Effective Complaint Submission**

#### **✅ Before Submitting**
1. **Gather all relevant information**
2. **Take photos/videos** if applicable
3. **Check if issue is already reported**
4. **Choose appropriate category and priority**

#### **✅ During Process**
1. **Respond promptly** to handler questions
2. **Provide additional information** when requested
3. **Stay professional** in all communications
4. **Track progress** regularly

#### **✅ After Resolution**
1. **Provide feedback** on the process
2. **Rate your satisfaction** with resolution
3. **Suggest improvements** for the system
4. **Help others** by sharing your experience

### **Best Practices**

#### **📝 Writing Effective Complaints**
- **Be specific** about dates, times, locations
- **Describe impact** on your studies/activities
- **Include evidence** (photos, documents)
- **Stay objective** and professional
- **Focus on solutions** rather than just problems

#### **🤝 Working with Handlers**
- **Be respectful** of their time and expertise
- **Provide clear information** they need
- **Follow up appropriately** without excessive messaging
- **Acknowledge good service** when provided
