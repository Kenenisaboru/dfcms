## 1. Project Overview
The **Digital Feedback & Complaint Management System (DFCMS)** is designed to transform paper-based grievances into actionable, real-time data objects. It ensures institutional accountability through a strictly enforced role-based escalation cycle: 
**Student → Class Representative → Teacher → HOD.**

---

## 2. Key Features
- **Modern UI/UX**: Premium glassmorphism design with high-contrast accessibility across all dashboards.
- **Hierarchical Routing**: Automatic and manual escalation based on institution protocols.
- **Audit Trails**: Non-repudiable logs of every action, forward, and resolution.
- **Evidence Management**: Secure file-upload engine for grievance-related attachments.
- **Real-Time Notifications**: Instant feedback loop for all involved roles.

---

## 3. Technology Stack
- **Frontend**: HTML5, CSS3 (Glassmorphism), Bootstrap 5.3, JavaScript (ES6+).
- **Backend**: PHP 8.2 (Secure PDO Implementation).
- **Database**: MySQL (InnoDB Engine).
- **Imagery**: Custom high-fidelity mockups generated for professional presentations.

---

## 4. Immediate Setup (Local)
1.  **Clone**: `git clone https://github.com/Kenenisaboru/dfcms.git`
2.  **Move**: Place in XAMPP `htdocs` directory.
3.  **Database**: Create `dfcms` in PHPMyAdmin and import `database.sql`.
4.  **Configure**: Update credentials in `config/database.php`.
5.  **Access**: Visit `http://localhost/dfcms/`.

## 5. System Roles & Architecture
![DFCMS Communication Flow Illustration](/C:/Users/Mame PC Clinic/.gemini/antigravity/brain/fba5d121-ecac-4f4c-9247-44336c68d838/dfcms_communication_flow_illustration_1775113358376.png)

### Role Responsibilities:
- **Student**: Grievance originator and lifecycle observer.
- **Representative (CR)**: Triage authority and minor resolution specialist.
- **Instructional (Teacher)**: Subject-matter investigation and lab task assignment.
- **Executive (HOD)**: Final institutional authority and strategic oversight.
